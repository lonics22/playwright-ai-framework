准备工作

就像准备做饭前要洗好菜，写代码也需要一些基础工具

📦 必备清单

Python 3.11+
我们的"主厨"，必须安装

代码编辑器（推荐 VS Code）
稳定的网络
下载东西要用

🔍 检查 Python 是否安装

Windows 用户：
  1. 按下 Win + R 键
  2. 输入 cmd 回车
  3. 输入以下命令：
python --version
Mac 用户：
  1. 打开"终端"（应用程序 → 实用工具）
  2. 输入命令：
python3 --version
成功标志：看到类似 Python 3.11.4 的字样就说明安装成功了！
提示：如果提示"找不到命令"，先去 python.org 下载安装 Python

下载框架代码

把"菜谱"拿回家，这一步就像去书店买食谱

📥 使用 Git 下载（推荐）

# 1. 先找个地方放代码（比如桌面）
cd Desktop

git clone https://github.com/your-username/playwright-ai-framework.git

📁 桌面/ └── 📁 playwright-ai-framework/ ← 你在这里 ├── 📁 config/ ├── 📁 core/ ├── 📁 pages/ ├── 📁 tests/ ├── 📁 utils/ ├── 📄 requirements.txt ├── 📄 pytest.ini └── 📄 README.md
没有 Git？ 直接去网页下载 ZIP 包，解压到桌面也可以！
3

创建虚拟环境

给项目建个"独立小屋"

为什么要这样做？

想象你在做蛋糕，需要特定品牌的面粉。虚拟环境就像给这个蛋糕专门准备的一个储物柜，里面放它需要的所有材料，不会和其他蛋糕的材料混在一起。

🏗️ 创建虚拟环境

# Windows
python -m venv .venv
# Mac
python3 -m venv .venv

⚡ 激活虚拟环境

# Windows
.venv\Scripts\activate
# Mac
source .venv/bin/activate
成功标志：你会看到命令行前面多了 (.venv) 字样
# 激活前：
C:\Users\你的名字\Desktop\playwright-ai-framework>

# 激活后：
(.venv) C:\Users\你的名字\Desktop\playwright-ai-framework>
4

安装依赖

准备食材 - 安装项目需要的各种工具包

📦 安装 Python 包

pip install -r requirements.txt

这个过程会安装什么？

🧪 pytest - 测试框架
🎭 playwright - 浏览器自动化
📊 allure-pytest - 报告工具
🤖 browser-use - AI 功能
🔗 langchain - AI 连接工具
📄 PyYAML - YAML 解析
等待时间：根据网络情况，大概需要 2-5 分钟。看到绿色文字 "Successfully installed" 就是成功了！

🌐 安装浏览器

Playwright 需要特定的浏览器版本：

playwright install chromium

这就像下载了一个特定版本的 Chrome 浏览器专门给自动化测试用

配置环境变量

设置"密码本" - 告诉程序一些秘密配置

📄 复制配置文件

# Windows
copy .env.example .env
# Mac
cp .env.example .env

🗂️ 文件位置

📁 playwright-ai-framework ├── 📄 .env ← 修改这个文件 └── 📄 .env.example

✏️ 编辑 .env 文件

用 VS Code 打开项目文件夹，找到 .env 文件，内容改成：

# API Keys for AI Testing（AI测试用的密钥）
# 如果你只想跑传统测试，可以先不管这些
OPENAI_API_KEY=sk-你的openai密钥
ANTHROPIC_API_KEY=sk-ant-你的anthropic密钥

# Test Environment（测试环境）
TEST_ENV=default

# Browser Configuration（浏览器配置）
BROWSER=headless
新手提示：如果你没有 OpenAI 密钥，没关系！可以先跑传统测试，不需要填这个。传统测试完全免费！
6

写第一个测试用例

做第一道菜！我们来测试打开百度首页

📄 创建测试文件

在 tests/traditional/ 文件夹下新建文件 test_my_first.py

📁 playwright-ai-framework ├── 📁 tests │ ├── 📁 traditional │ │ └── 📄 test_my_first.py ← 新建这个 │ └── ...

📝 写入测试代码

# tests/traditional/test_my_first.py
import allure
from pages.base_page import BasePage

@allure.feature("我的第一个测试")
class TestMyFirst:
    """这是注释：新手入门测试"""

    @allure.title("访问百度首页")
    def test_visit_baidu(self, page):
        """测试打开百度首页"""
        # 1. 创建页面对象（拿起锅铲）
        base_page = BasePage(page)

        # 2. 打开百度（开火炒菜）
        base_page.navigate_to("https://www.baidu.com")

        # 3. 等待页面加载完成（等待菜熟）
        base_page.wait_for_page_load()

        # 4. 验证页面标题包含"百度"（尝一口看味道对不对）
        assert "百度" in base_page.get_page_title()

        # 5. 截图留念（拍照发朋友圈）
        base_page.take_screenshot("百度首页")

🎯 测试代码就像做一道菜的步骤：

1
准备工具（import + 创建对象）
2
开火下锅（navigate_to 打开网页）
3
等待煮熟（wait_for_page_load）
4
尝味道（assert 断言验证）
5
拍照记录（take_screenshot）

尝试 AI 测试（进阶）

体验 AI 的神奇之处

🤖 AI 测试有什么不同？

传统测试：

你告诉电脑：点击 id='kw' 的元素，输入"Python教程"

AI 测试：

你告诉电脑：在搜索框输入"Python教程"
电脑自己找搜索框！

📝 AI 测试示例代码

import pytest
import allure

@pytest.mark.ai
@allure.feature("AI 演示测试")
class TestAIDemo:

    @allure.title("AI 自动搜索")
    async def test_ai_search(self, async_ai_agent, async_page):
        """让 AI 自动完成搜索任务"""
        # 打开百度
        await async_page.goto("https://www.baidu.com")

        # 让 AI 执行搜索（你只需要说一句话！）
        result = await async_ai_agent.execute_task(
            "在搜索框输入'Python教程'并点击搜索按钮"
        )

        # 验证成功
        assert result.success
# 运行 AI 测试
pytest tests/ai_driven/test_ai_demo.py -v -m ai

🔧 常见问题排查

Q1: 提示 "pip 不是内部或外部命令"

原因：Python 安装时没勾选 "Add to PATH"

解决：重新安装 Python，安装时勾选 "Add Python to PATH"

Q2: 提示 "playwright 不是内部或外部命令"

原因：虚拟环境没激活

解决：重新激活虚拟环境

Q3: 浏览器启动失败

原因：playwright 浏览器没装好

解决：运行 playwright install chromium

Q4: 测试运行时浏览器一闪而过

这是正常的！ 想看浏览器过程，在 .env 中设置：BROWSER=headed
